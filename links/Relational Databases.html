Relational Databases

I experience relational databases mostly through SQL language ("Structured Query Language") which will interact with the database for my dynamic site.


CREATE DATABASE Query create a new, and empty database within the database management system, essentially sets up a new container to hold tables, data, and other database objects.


CREATE TABLE Query defines a new table with specified columns, data types, rules, and used for structuring the database.


INSERT INTO Query adds new rows of data into a table, essential for storing new information.


SELECT Query retrieves specific columns from a table, filtering rows based on a condition to get the data.


UPDATE Query modifies existing data in a table based on a specified condition, used for keeping information updated.


DELETE Query removes rows from a table that meet a certain condition, necessary for removing outdated or incorrect data.


ALTER TABLE Query modifies an existing table structure, useful for evolving the database schema.


DROP TABLE Query removes an entire table and its data.


DROP DATABASE Query permanently deletes the specified database and all the tables, and data.
